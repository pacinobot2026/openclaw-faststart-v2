## Chrome Extension Install Fallback - BOOTSTRAP.md Fix

**Time:** ~8:00 AM CST

### Problem
Customer VPS ran `openclaw browser extension install` but got error:
```
Error: Bundled Chrome extension is missing. Reinstall OpenClaw and try again.
```

The OpenClaw npm package on customer machine was missing `assets/chrome-extension` folder. Still unknown WHY (installation mystery - script didn't have npm install command but OpenClaw still got installed somehow).

### Solution
Added fallback to BOOTSTRAP.md Step 4. If official command fails, download extension directly from GitHub:

```powershell
$d="$env:USERPROFILE\.openclaw\browser\chrome-extension"
$i="$d\icons"
New-Item -ItemType Directory -Force -Path $i | Out-Null
$b="https://raw.githubusercontent.com/openclaw/openclaw/main/assets/chrome-extension"
@("manifest.json","background.js","options.html","options.js","README.md") | ForEach-Object { curl.exe -s -o "$d\$_" "$b/$_" }
@("icon16.png","icon32.png","icon48.png","icon128.png") | ForEach-Object { curl.exe -s -o "$i\$_" "$b/icons/$_" }
Write-Host "Chrome extension downloaded to: $d"
```

### Key Learnings
- Extension files are at: `assets/chrome-extension` in npm package
- Official install copies to: `~\.openclaw\browser\chrome-extension`
- GitHub raw URL: `https://raw.githubusercontent.com/openclaw/openclaw/main/assets/chrome-extension/`
- download-directory.github.io doesn't work with curl (JS-based, generates zip client-side)
- "Browser control enabled" just means Gateway is running (relay server auto-starts on port 18792)

## ROOT CAUSE FOUND - Chrome Extension Missing

**The mystery is solved:**

The VPS setup script has this:
```powershell
# Example:
iwr -useb https://openclaw.ai/install.ps1 | iex
```

The `iwr` command is on its OWN LINE (not commented!) - so OpenClaw DOES get installed via the official `https://openclaw.ai/install.ps1` installer.

**The problem:** That official install script doesn't bundle the chrome extension assets.

**So:**
- OpenClaw binary ✅ installed
- Chrome extension ❌ missing

**Our fix:** BOOTSTRAP.md Step 4 fallback downloads extension from GitHub when bundled is missing.

**Real fix would be:** OpenClaw team updates their install.ps1 to include the extension. Could report this to them.
