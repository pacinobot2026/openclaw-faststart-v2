## Course Sprout Lesson Creation Workflow (MEMORIZED)

**Tested on:** Pacino Test (Course ID: 349)

### The Process
1. **Get course ID** вЂ” `GET /get-course` to find or confirm course
2. **Create lesson with video** вЂ” `POST /create-lesson` with ALL fields:
   - `name` вЂ” Lesson title
   - `course_id` вЂ” Target course
   - `short_description` вЂ” Brief summary
   - `long_description` вЂ” Full content (NO emojis - API rejects them)
   - `video_name` вЂ” Name for video library
   - `video_url` вЂ” **FULL Vimeo URL** (e.g., `https://vimeo.com/1164151348`)
   - `video_tags` вЂ” Array like `["claw"]` to organize by course

### Key Learnings
- **Full Vimeo URL required** вЂ” not just the ID
- **No emojis** in descriptions вЂ” causes 422 error
- **Video tags = course identifier** вЂ” all videos in a course share same tag (e.g., "claw" for OpenClaw content)
- **No update endpoint** вЂ” each create makes a new lesson, delete duplicates manually
- **Existing videos reused** вЂ” if video_url matches, reuses existing (tags won't update)
- **For fresh video with tags** вЂ” need unique video or delete existing first

### Example API Call
```json
{
  "name": "Lesson Title",
  "course_id": 349,
  "short_description": "Brief summary",
  "long_description": "Full content here...",
  "video_name": "Video Library Name",
  "video_url": "https://vimeo.com/1164151348",
  "video_tags": ["claw"]
}
```

## 1:58 PM - Goal Block Framework (MEMORIZED)

**Chad's Course Sprout Goal Block Intelligence Engine**

### Classification First
- **Tactical/Software** ? 4-8 execution steps
- **Strategic Framework** ? Planning exercise tied to their business
- **Conceptual/Mindset** ? Reflection, epiphany prompts, future pacing
- **Hybrid** ? Both steps and reflection

### The Rules
1. **NO homework vibes** пїЅ Keep it FUN
2. **5th grade reading level** пїЅ Simple language
3. **Just a COUPLE things to do** пїЅ Not a to-do list
4. **NO forecasting next lesson** пїЅ That's Chad's job
5. **NO fabricating** пїЅ Only reference what was actually taught
6. **NO generic fluff** пїЅ Specific to the lesson content

### The Structure
1. **"Here's the deal"** пїЅ Reinforce the big breakthrough they just learned
2. **"Your one thing"** пїЅ ONE simple action (couple of ideas max)
3. **"Drop a comment below"** пїЅ Engagement prompt
4. **Positive reinforcement** пїЅ Questions that MATCH what they just did/learned

### Example (Newsletter 2.0 Lesson):
```
Here's the deal:
You just learned something that changes EVERYTHING.
Articles aren't just blog posts. They're little traffic machines...
One piece of content. Two jobs. Way less work.

Your one thing:
Think of 3 things happening in your town right now that people might Google.
That's it. Just 3 ideas.

Now drop a comment below!
Did it click for you? Can you see how one article does TWO jobs now? 
What's the first topic YOU want to write about in your town?
```

### Key Insight
Reinforcement questions must be CONSISTENT with what they just learned.
- Generic: "Are you loving the course?" ?
- Specific: "Can you see how one article does TWO jobs now?" ?
