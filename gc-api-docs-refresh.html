Global Control API

# Global Control API The Global Control API provides programmatic access to your CRM data — contacts, tags, workflows, domains, email broadcasts, and more. ## Base URL ``` https://api.globalcontrol.io/api/ai ``` ## Response Format All responses use the standard wrapper: ```json { "type": "response", "data": { } } ``` List endpoints include pagination and status counters inside `data`.

Authentication

All API requests require an API key sent in the `X-API-KEY` header. > `X-API-KEY: your-api-token-here`

https://api.globalcontrol.io/api/ai/

# Update a Workflow Multi-purpose endpoint that updates any aspect of a workflow. All fields are optional — send only the fields you want to change. The server accepts any field from the Workflow schema with no input validation. ## AI Model Instructions ### Managing Flows Flows can be **added**, **reordered**, or **deleted** through this endpoint. Before making any flow changes: 1. **Always call `GET /workflows/{id}` first** to retrieve the current workflow and understand its existing flow structure. 2. Review the returned `flows` array — check each flow's `type`, `index`, `label`, and `data`. 3. Ask the user exactly what they want: add a new flow, modify an existing flow's data, reorder steps, or delete a step. 4. Use the **Flow Data Structures** section below to prompt the user for the correct fields for the chosen flow type. 5. When adding a new flow, include it in the `flows` array with a new `index` (omit `_id` — server generates it). When deleting, omit it from the array.
