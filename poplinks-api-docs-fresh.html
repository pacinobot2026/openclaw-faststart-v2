MintBird API

MintBird API Collection with api_key authentication. ## Authentication All endpoints require `Authorization: Bearer {{api_key}}` header. ## Base URL `{{base_url}}/api/ai` ## Common Error Responses ### 401 Unauthorized ```json { "success": false, "message": "Unauthorized" } ``` ### 404 Not Found ```json { "success": false, "message": "Resource not found." } ``` ### 422 Validation Error ```json { "success": false, "message": { "field": ["The field is required."] } } ``` ### 500 Server Error ```json { "success": false, "message": "Server error: ..." } ```

Authentication

All API requests require an `api_key` provided in the header. > `Authorization: Bearer {{api_key}}`

POST

Create Poplink

POST https://api.poplinks.io/api/ai/poplinks
Create a new poplink (tracked redirect link). The poplink's status is auto-calculated based on `start_date` and `end_date`: - If `start_date` is in the future: status = `2` (Pending) - If `end_date` is in the past: status = `3` (Expired) - Otherwise: status = `1` (Active) ### Request Body Parameters | Parameter | Type | Required | Description | |-----------|------|----------|-------------| | `name` | string | Yes | Link name (max 255 chars) | | `destination_url` | string (URL) | Yes | Target URL. Must be a valid URL format. | | `visible_url` | string | Yes | URL slug (max 255 chars). Spaces are automatically removed. Must be unique on the chosen domain. | | `domain_id` | integer/string | Yes | Personal domain ID (integer) or system domain URL (string). Use `Get Personal Domains` or `Get System Domains` to find values. | | `domain_type` | string | No | `"personal"` or `"system"`. Default: `"personal"` | | `group_id` | integer | No | Group ID. Use `Get Groups` endpoint to find IDs. | | `vendor_id` | integer | No | Vendor ID. Use `Get Vendors` endpoint to find IDs. | | `isAdvanced` | integer | No | Enable advanced mode: `1` (yes) or `0` (no). Default: `0` | | `is_clocked` | integer | No | Enable clocked mode: `1` (yes) or `0` (no). Default: `0` | | `start_date` | string | No | Start date (YYYY-MM-DD). Link becomes active on this date. | | `end_date` | string | No | End date (YYYY-MM-DD). Link expires after this date. | ### Validation Rules - `visible_url` must be unique across poplinks, funnels, links, and snippets on the same domain - If `domain_type` is `"system"`, `domain_id` must be a valid system domain URL - If `domain_type` is `"personal"`, `domain_id` must be an existing domain ID belonging to the user
Request Body 
{
    "name": "My New Poplink",
    "destination_url": "https://example.com/offer-page",
    "visible_url": "my-offer",
    "domain_id": 1,
    "domain_type": "personal",
    "group_id": null,
    "vendor_id": null,
    "isAdvanced": 0,
    "is_clocked": 0,
    "start_date": null,
    "end_date": null
}
Example Response 201 Created 
{
    "success": true,
    "message": "Poplink created successfully.",
    "data": {
        "poplink": {
            "id": 10,
            "name": "My New Poplink",
            "destination_url": "https://example.com/offer-page",
            "visible_url": "my-offer",
            "domain_id": 1,
            "domain_type": "personal",
            "group_id": null,
            "vendor_id": null,
            "status": 1,
            "user_id": 42,
            "created_at": "2025-06-01T10:00:00.000000Z",
            "updated_at": "2025-06-01T10:00:00.000000Z"
        }
    }
}
Example Response 422 Unprocessable Entity 
{
    "success": false,
    "message": {
        "name": [
            "The name field is required."
        ],
        "destination_url": [
            "The destination url field is required."
        ]
    }
}
Example Response 422 Unprocessable Entity 
{
    "success": false,
    "message": "This URL slug is already in use on this domain."
}
Example Response 404 Not Found 
{
    "success": false,
    "message": "Domain not found or does not belong to you."
}
